# Compiler Passes Overview

microMLC provides six custom MLIR compiler passes specifically designed to optimize transformer workloads. These passes transform high-level StableHLO operations into efficient, vectorized machine code through a series of progressive lowering and optimization stages.

## Pass Architecture

All pass implementations are located in the [`passes/`](https://github.com/sakshambatra1/microMLC/tree/main/passes) directory:

## Pass Registration

Passes are registered in the StableHLO dialect through three key files:

### Passes.td (TableGen Definition)

```tablegen Passes.td
def FixUnrealizedIndexCastsPass : Pass<"fix-unrealized-index-casts", "ModuleOp"> {
  let summary = "Rewrite unrealized_conversion_cast iN -> index into arith.index_cast";
  let dependentDialects = ["mlir::arith::ArithDialect"];
}

def CfIntegerizeIndexBlockArgsPass
    : Pass<"cf-integerize-index-block-args", "ModuleOp"> {
  let summary = "Rewrite CF block args of type index to i64 and update branches";
  let dependentDialects = [
    "mlir::arith::ArithDialect",
    "mlir::cf::ControlFlowDialect",
    "mlir::func::FuncDialect"
  ];
  let constructor = "mlir::stablehlo::createCfIntegerizeIndexBlockArgsPass()";
}
```

### Passes.h (Header)

```cpp Passes.h
namespace mlir {
namespace stablehlo {

// Factory methods
std::unique_ptr<Pass> createGenericFusionPass();
std::unique_ptr<Pass> createToyVectorizationPass();
std::unique_ptr<Pass> createFixUnrealizedIndexCastsPass();
std::unique_ptr<Pass> createCfIntegerizeIndexBlockArgsPass();

// Registration methods
void registerGenericFusionPass();
void registerToyVectorizationPass();
void registerOneShotBufferizePass();

}  // namespace stablehlo
}  // namespace mlir
```

### CMakeLists.txt (Build System)

```cmake CMakeLists.txt
add_mlir_library(StablehloTransforms
  TilingPass.cpp
  GenericFusionPass.cpp
  LinalgTilingPass.cpp
  OneShotBufferizePass.cpp
  ToyVectorizationPass.cpp
  ParallelSchedulingPassAttn.cpp
  CfIntegerizeIndexBlockArgsPass.cpp
  FixUnrealizedIndexCastsPass.cpp
  # ... dependencies
)
```

## Pass Pipeline

The standard optimization pipeline applies passes in this order:

<Steps>
  <Step title="StableHLO Tiling">
    Convert StableHLO dot_general operations to tiled Linalg loops
    ```bash
    -stablehlo-tiling
    ```
  </Step>
  
  <Step title="Generic Fusion">
    Fuse producer-consumer chains of Linalg generic operations
    ```bash
    -generic-fusion
    ```
  </Step>
  
  <Step title="Multi-Level Tiling">
    Apply L2 (64×64×64) and L1 (4×4×4) cache-aware tiling
    ```bash
    -stablehlo-linalg-tiling=l2-tile-sizes=64,64,64 l1-tile-sizes=4,4,4
    ```
  </Step>
  
  <Step title="Bufferization">
    Convert tensor operations to memref with explicit memory management
    ```bash
    -empty-tensor-to-alloc-tensor -one-shot-bufferize -convert-bufferization-to-memref
    ```
  </Step>
  
  <Step title="Vectorization">
    Generate SIMD vector instructions (width 4 for AVX2)
    ```bash
    -toy-vectorize=vector-width=4 enable-reductions
    ```
  </Step>
</Steps>

## Pass Dependencies

Passes depend on various MLIR dialects:

| Pass | Key Dialects |
|------|-------------|
| TilingPass | StableHLO, Linalg, SCF, Tensor, Arith |
| GenericFusionPass | Linalg, Tensor, Arith, SCF |
| LinalgTilingPass | Linalg, SCF, Tensor |
| OneShotBufferizePass | Bufferization, Memref, Arith |
| ToyVectorizationPass | Vector, Linalg, Memref, Arith |
| ParallelSchedulingPass | SCF, Affine, Async |

## Using Custom Passes

### Command-Line Execution

Run passes with `stablehlo-opt`:

```bash
third_party/stablehlo/build/bin/stablehlo-opt \
  input.mlir \
  -stablehlo-tiling \
  -generic-fusion \
  -o output.mlir
```

### Pipeline Script

Use the automated pipeline:

```bash
bash tools/run_attn_cache_pipeline.sh
```

This executes all five stages and writes intermediate IR to `ir/attn_stage*.mlir`.

## Development Workflow

To modify or add passes:

<Steps>
  <Step title="Edit pass implementation">
    Modify the `.cpp` file in `passes/` directory
  </Step>
  
  <Step title="Sync to StableHLO">
    ```bash
    bash tools/sync_passes.sh
    ```
    Copies your changes to `third_party/stablehlo/stablehlo/transforms/`
  </Step>
  
  <Step title="Rebuild stablehlo-opt">
    ```bash
    cmake --build third_party/stablehlo/build -t stablehlo-opt
    ```
  </Step>
  
  <Step title="Test your changes">
    ```bash
    bash tools/run_attn_cache_pipeline.sh
    ```
  </Step>
</Steps>

<Note>
  The `sync_passes.sh` script copies 12 files from your local `passes/` directory to the StableHLO build tree. Always run this after modifying pass code.
</Note>

## Performance Impact

Each pass contributes to overall performance improvement:

| Stage | Pass | Lines of IR | Speedup |
|-------|------|-------------|----------|
| 0 | StableHLO (baseline) | 84 | 1.0× |
| 1 | + Fusion | 88 | 1.1× |
| 2 | + Multi-Level Tiling | 17,731 | 1.3× |
| 3 | + Bufferization | 17,961 | 1.5× |
| 5 | + Vectorization | 20,306 | **2.5×** |

<Tip>
  The vectorization pass provides the largest performance gain by enabling SIMD execution on modern CPUs.
</Tip>
