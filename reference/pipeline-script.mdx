## Overview

The `run_attn_cache_pipeline.sh` script implements microMLC's core compilation strategy: a five-stage progressive lowering pipeline that transforms StableHLO attention IR into optimized, vectorized memref code.

## Location

```
tools/run_attn_cache_pipeline.sh
```

## Purpose

This pipeline demonstrates a **split optimization strategy** where high-level transformations (fusion, tiling) are applied on tensors before bufferization, followed by low-level optimizations (vectorization) on memrefs. This approach keeps the IR in SSA form longer, enabling better analysis and optimization.

---

## Pipeline Stages

```mermaid
graph LR
    A[StableHLO Input] --> B[Stage 0: Linalg]
    B --> C[Stage 1: Fused]
    C --> D[Stage 2: Tiled]
    D --> E[Stage 3: Bufferized]
    E --> F[Stage 5: Vectorized]
```

### Stage 0: Linalg Conversion

**Input**: `ir/attn_complete.mlir` (StableHLO)

**Output**: `ir/attn_stage0_linalg.mlir`

**Passes Applied**:
```bash
-stablehlo-tiling
-canonicalize -cse
```

**What Happens**:
- Converts StableHLO ops to named linalg operations (`linalg.matmul`, `linalg.generic`)
- Preserves high-level structure (batch matmuls, softmax reduction)
- Maintains tensor semantics

**Key Operations**:
```mlir
%10 = linalg.batch_matmul ins(%q, %k : tensor<4x128x32xf32>, tensor<4x32x128xf32>)
                          outs(%scores : tensor<4x128x128xf32>) -> tensor<4x128x128xf32>

%14 = linalg.generic {indexing_maps = [#map3, #map4], 
                      iterator_types = ["parallel", "parallel", "reduction"]} 
      ins(%scaled : tensor<4x128x128xf32>) outs(%max : tensor<4x128xf32>) {
  ^bb0(%in: f32, %acc: f32):
    %max_val = arith.cmpf ogt, %in, %acc : f32
    %sel = arith.select %max_val, %in, %acc : f32
    linalg.yield %sel : f32
} -> tensor<4x128xf32>
```

---

### Stage 1: Epilogue Fusion

**Input**: Stage 0 output

**Output**: `ir/attn_stage1_fused.mlir`

**Passes Applied**:
```bash
-generic-fusion
-canonicalize -cse
```

**What Happens**:
- Fuses elementwise epilogues (scale, exp, div) into their producers
- Reduces intermediate allocations
- Creates multi-input generic ops

**Example Fusion**:

**Before** (Stage 0):
```mlir
%scaled = linalg.generic ... {  // scale
  %res = arith.mulf %in, %cst_scale : f32
  linalg.yield %res : f32
}
%exp = linalg.generic ... {  // exp (separate op)
  %sub = arith.subf %in, %max : f32
  %res = math.exp %sub : f32
  linalg.yield %res : f32
}
```

**After** (Stage 1):
```mlir
%exp = linalg.generic ... {  // fused scale+exp
  %scaled = arith.mulf %in, %cst_scale : f32
  %sub = arith.subf %scaled, %max : f32
  %res = math.exp %sub : f32
  linalg.yield %res : f32
}
```

### Stage 2: Two-Level Tiling

**Input**: Stage 1 output

**Output**: `ir/attn_stage2_tiled_tensor.mlir`

**Passes Applied**:
```bash
"-stablehlo-linalg-tiling=l2-tile-sizes=64,64,64 l1-tile-sizes=4,4,4 enable-promotion=false"
-canonicalize -cse
```

**What Happens**:
- Tiles matmuls into nested loops with two tile sizes
- **L2 tile (64×64×64)**: Fits in L2 cache (~256KB)
- **L1 tile (4×4×4)**: Fits in L1 cache, enables vectorization
- **No promotion**: Avoids explicit cache management (relies on hardware)

**Tiling Strategy**:
```mlir
// Original matmul: (4, 128, 128) @ (4, 128, 32) -> (4, 128, 32)

scf.for %i = 0 to 128 step 64 {        // L2 tile
  scf.for %j = 0 to 32 step 64 {
    scf.for %k = 0 to 128 step 64 {
      %tile_A = tensor.extract_slice %A[%b, %i, %k] [1, 64, 64] [1, 1, 1]
      %tile_B = tensor.extract_slice %B[%b, %k, %j] [1, 64, 64] [1, 1, 1]
      
      scf.for %ii = 0 to 64 step 4 {   // L1 tile
        scf.for %jj = 0 to 64 step 4 {
          scf.for %kk = 0 to 64 step 4 {
            %micro_A = tensor.extract_slice %tile_A[0, %ii, %kk] [1, 4, 4] [1, 1, 1]
            %micro_B = tensor.extract_slice %tile_B[0, %kk, %jj] [1, 4, 4] [1, 1, 1]
            %micro_C = linalg.matmul ins(%micro_A, %micro_B) ...  // 4×4×4 kernel
          }
        }
      }
    }
  }
}
```

**Why L1=4?**
- Enables 4-wide SIMD vectors (128-bit registers hold 4× f32)
- Small enough for register allocation
- Matches typical NEON/SSE/AVX vector widths

---

### Stage 3: Bufferization

**Input**: Stage 2 output

**Output**: `ir/attn_stage3_bufferized.mlir`

**Passes Applied**:
```bash
-canonicalize -cse
-empty-tensor-to-alloc-tensor
"-one-shot-bufferize=bufferize-function-boundaries allow-return-allocs"
-convert-bufferization-to-memref
-canonicalize -cse
```

**What Happens**:
- Converts tensors to memrefs (SSA → imperative)
- Allocates memory for intermediate results
- Inserts explicit stores instead of tensor assignments

**Transformation Example**:

**Before** (Tensor):
```mlir
%result = linalg.matmul ins(%A, %B : tensor<4x4xf32>, tensor<4x4xf32>)
                        outs(%C : tensor<4x4xf32>) -> tensor<4x4xf32>
return %result : tensor<4x4xf32>
```

**After** (Memref):
```mlir
%A_mem = memref.alloc() : memref<4x4xf32>
%B_mem = memref.alloc() : memref<4x4xf32>
%C_mem = memref.alloc() : memref<4x4xf32>
linalg.matmul ins(%A_mem, %B_mem : memref<4x4xf32>, memref<4x4xf32>)
              outs(%C_mem : memref<4x4xf32>)
return %C_mem : memref<4x4xf32>
```

**Function Boundaries**:
- `bufferize-function-boundaries`: Converts function signatures
- `allow-return-allocs`: Permits returning allocated memrefs (caller deallocates)

---

### Stage 5: Vectorization

**Input**: Stage 3 output

**Output**: `ir/attn_stage5_vectorized.mlir`

**Passes Applied**:
```bash
"-toy-vectorize=vector-width=4 enable-reductions"
-canonicalize -cse
```

**What Happens**:
- Replaces scalar loops with vector operations
- Uses 4-wide SIMD (matches L1 tile size)
- Vectorizes both compute and reductions

**Vectorization Example**:

**Before** (Scalar):
```mlir
scf.for %i = 0 to 4 step 1 {
  %a = memref.load %A[%i] : memref<4xf32>
  %b = memref.load %B[%i] : memref<4xf32>
  %c = arith.mulf %a, %b : f32
  memref.store %c, %C[%i] : memref<4xf32>
}
```

**After** (Vector):
```mlir
%va = vector.transfer_read %A[%c0], %pad : memref<4xf32>, vector<4xf32>
%vb = vector.transfer_read %B[%c0], %pad : memref<4xf32>, vector<4xf32>
%vc = arith.mulf %va, %vb : vector<4xf32>
vector.transfer_write %vc, %C[%c0] : vector<4xf32>, memref<4xf32>
```

**Reduction Vectorization**:
```mlir
// Horizontal sum reduction
%acc = vector.transfer_read %init[%c0], %zero : memref<f32>, vector<4xf32>
scf.for %i = 0 to 128 step 4 {
  %v = vector.transfer_read %data[%i], %zero : memref<128xf32>, vector<4xf32>
  %acc = arith.addf %acc, %v : vector<4xf32>
}
%sum = vector.reduction <add>, %acc : vector<4xf32> into f32
```

## Script Source

```bash
#!/usr/bin/env bash
set -euo pipefail
set -x

OPT_BIN="third_party/stablehlo/build/bin/stablehlo-opt"
IR_DIR="ir"
INPUT="${IR_DIR}/attn_complete.mlir"

OUT0="${IR_DIR}/attn_stage0_linalg.mlir"
OUT1="${IR_DIR}/attn_stage1_fused.mlir"
OUT2="${IR_DIR}/attn_stage2_tiled_tensor.mlir"
OUT3="${IR_DIR}/attn_stage3_bufferized.mlir"
OUT5="${IR_DIR}/attn_stage5_vectorized.mlir"

[[ -x "${OPT_BIN}" ]] || { echo "stablehlo-opt not found at ${OPT_BIN}"; exit 1; }
mkdir -p "${IR_DIR}"
[[ -f "${INPUT}" ]] || { echo "Input MLIR missing at ${INPUT}"; exit 1; }

# Stage 0: initial stablehlo cleanup
"${OPT_BIN}" \
  -stablehlo-tiling \
  -canonicalize -cse \
  "${INPUT}" -o "${OUT0}"

# Stage 1: fuse epilogues while keeping named matmuls
"${OPT_BIN}" \
  -generic-fusion \
  -canonicalize -cse \
  "${OUT0}" -o "${OUT1}"

# Stage 2: tensor tiling (no promotion) with L1=4 for small vectors
"${OPT_BIN}" \
  "-stablehlo-linalg-tiling=l2-tile-sizes=64,64,64 l1-tile-sizes=4,4,4 enable-promotion=false" \
  -canonicalize -cse \
  "${OUT1}" -o "${OUT2}"

# Stage 3: bufferize to memrefs
"${OPT_BIN}" \
  -canonicalize -cse \
  -empty-tensor-to-alloc-tensor \
  "-one-shot-bufferize=bufferize-function-boundaries allow-return-allocs" \
  -convert-bufferization-to-memref \
  -canonicalize -cse \
  "${OUT2}" -o "${OUT3}"

# Stage 5: vectorize directly after bufferization
"${OPT_BIN}" \
  "-toy-vectorize=vector-width=4 enable-reductions" \
  -canonicalize -cse \
  "${OUT3}" -o "${OUT5}"

echo "Pipeline completed. Outputs:"
echo "  ${OUT0}"
echo "  ${OUT1}"
echo "  ${OUT2}"
echo "  ${OUT3}"
echo "  ${OUT5}"
```

## Design Rationale

### Why Split Optimization?

**Tensor-level** (Stages 0-2):
- Fusion sees def-use chains clearly
- Tiling operates on pure functions (no aliasing)
- Analysis is simpler (SSA form)

**Memref-level** (Stages 3-5):
- Vectorization needs explicit memory layout
- Hardware mapping requires allocation control
- Code generation works on imperative IR

### Why No Stage 4?

Original experiments included a "parallel scheduling" stage, but it was removed because:
- OpenMP pragmas added overhead for small kernels
- Vectorization (Stage 5) provided better speedups
- CPU-level parallelism better left to framework (JAX, TVM)

### Why L2=64, L1=4?

**L2 cache targeting** (64×64 tiles):
- 64×64×4 bytes = 16KB per tile
- 3 tiles (Q, K, V) = 48KB
- Fits comfortably in ~256KB L2

**L1 cache targeting** (4×4 tiles):
- 4×4×4 bytes = 64 bytes per tile
- Fits in single cache line
- Enables 4-wide SIMD (SSE/NEON)

### Promotion Disabled

```bash
enable-promotion=false
```

Promotion copies tiles to explicitly managed buffers (like scratchpad). Disabled because:
- Modern CPUs have effective hardware prefetching
- Explicit copies add overhead
- Compiler can't always prove buffer non-aliasing

## Troubleshooting

### Pass Not Found

```
Error: unknown pass name 'generic-fusion'
```

**Solution**: Run `./tools/sync_passes.sh` and rebuild StableHLO.

### Input File Missing

```
Input MLIR missing at ir/attn_complete.mlir
```

**Solution**: Generate IR from Python model:
```bash
cd transformer/python
python gen_mhattn_ir.py
cp attn_unmasked.mlir ../../ir/attn_complete.mlir
```

### Segfault During Execution

If benchmark scripts crash:
- Check LLVM version (requires 17)
- Verify OpenMP library is available
- Reduce iteration count: `ITERS=10`
